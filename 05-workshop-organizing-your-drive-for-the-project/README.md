# Workshop: organizing your Drive for the project

THEORETICAL RECAP Google Drive is a cloud-based file storage and collaboration platform that allows you to create, organize, share, and access files from any device. In a professional digital marketing context, a well-structured Drive is not a luxury — it is a foundational operational tool. Think of it as your digital office: if your desk is chaotic, your work will be chaotic too. The key mental model to adopt is the hierarchical folder structure: one main project folder at the root level, then subfolders organized by theme or function (strategy, content, analytics, tools, client assets, etc.), and files clearly named with a consistent convention (e.g., YYYY-MM-DD_filename_version). This structure mirrors how real marketing teams work in agencies and companies. Common mistakes beginners make: saving everything at the root level without folders, using vague file names like 'document1' or 'final_FINAL2', mixing personal and professional files, and never using Google Docs or Sheets natively (uploading Word/Excel files instead). These habits create confusion, slow you down, and make collaboration impossible. Mini example: A freelance digital marketer working on a client campaign would typically have a folder called 'Client_X_Campaign_2024', with subfolders: 'Brief', 'Strategy', 'Content', 'Analytics', 'Deliverables'. Each file is named clearly and dated.
